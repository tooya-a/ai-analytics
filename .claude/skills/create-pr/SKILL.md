# PR作成

会話・差分からPRタイトル・本文を自動生成し、`gh pr create` でGitHubにPRを作成する。

## テンプレート参照

`references/01-pr-template.md` を本文の骨格として使用する。

## 手順

### 1. 環境確認

```bash
which gh && gh --version
gh auth status
```

- `gh` が見つからない場合 → `brew install gh` を案内して終了
- 未認証の場合 → `gh auth login` を案内して終了

### 2. ブランチ・差分を収集

```bash
git branch --show-current
git log main..HEAD --oneline
git diff main..HEAD --stat
```

### 3. PRタイトル・本文を生成

- **タイトル形式**: `[カテゴリ] 変更内容を一行で`
  - カテゴリ例: `feat` / `fix` / `refactor` / `docs` / `chore`
  - 例: `[fix] ログ0件時の空状態表示に対応`
- **本文**: `01-pr-template.md` の各セクションをコミット・差分・会話の内容から埋める
  - `## 概要` → 変更内容を箇条書き（コミットメッセージ・差分から）
  - `## 変更点` → 会話から課題・背景・変更の意図を抽出して記述
  - `## 動作手順` → この変更を確認・再現するための手順を記述
  - `## 確認事項` → 固定チェックリストをそのまま使用

### 4. 内容確認

生成したタイトルと本文を表示して、以下を聞く：

> この内容でよいですか？　**「はい」で作成します。**

- 「はい」→ PR作成へ
- 修正依頼があれば対応してから再確認する

### 5. PR作成

```bash
gh pr create --title "タイトル" --body "$(cat <<'EOF'
本文
EOF
)"
```

作成されたPR URLを表示して完了。

## 注意事項

- `main` ブランチで実行した場合はfeatureブランチへの切り替えを促す
- コミットが0件の場合は「変更をコミットしてから実行してください」と案内して終了
- PRが既に存在する場合は `gh pr view` のURLを表示して終了
