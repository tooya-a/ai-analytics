# タスクレポート生成

## 概要

現在の会話セッションを分析し、作業レポートをGoogle Driveに保存する。

## 手順

### 1. セッション情報の収集

会話履歴から以下を分析する：

- **セッション開始・終了時刻**: 最初と最後のメッセージのタイムスタンプ（不明な場合は今日の日付を使用）
- **メッセージ数**: ユーザーとアシスタントのやり取り総数
- **工数推定**: メッセージ数と会話の密度から推定（例：「約1.5時間、会話20往復」）

### 2. サマリー生成

会話全体を振り返り、以下の項目を整形する：

**完了タスク**
- 実装・修正・設定した内容を箇条書きで列挙

**変更ファイル**
- `git diff --name-only HEAD` または会話中で言及・編集したファイルを列挙

**決定事項・メモ**
- 設計判断、方針転換、将来への申し送り事項

**未完了・次回タスク**
- 途中で止まった作業、次回持ち越しの項目

### 3. レポートフォーマット

以下の形式でドキュメント本文を組み立てる：

```
# 作業レポート - {日付}

## セッション情報
- 日付: {YYYY-MM-DD}
- 推定作業時間: {X時間Y分}
- 会話数: {N往復}
- ブランチ: {git branch}

## 完了タスク
{箇条書き}

## 変更ファイル
{ファイルリスト}

## 決定事項・メモ
{箇条書き}

## 今日学んだこと
{作業中に気づいた知見・ハマりポイントと解決策・新しく理解したことなど}

## スキルの作成手順のコツ
{スキル・設定・ツール作成で得た再現性のあるノウハウ}
※ セッション中にスキルや設定ファイルを作成した場合のみ記載

## 次回タスク
{箇条書き}
```

### 4. Google Driveへの保存

`mcp__claude_ai_Google_Drive__create_file` を使用してドキュメントを作成する：

- **title**: `作業レポート - {YYYY-MM-DD}` （引数があればそれをサフィックスに追加）
- **mimeType**: `text/plain`（Google Docsに自動変換される）
- **content**: 上記フォーマットのテキストをbase64エンコードして渡す

```
base64エンコード例（Node.js）:
Buffer.from(reportText).toString('base64')
```

保存後、返却されたファイルのURLまたはIDをユーザーに表示する。

### 5. 完了通知

- Google DriveのドキュメントURLを表示
- 「TODOS.md の完了済みタスクを更新しますか？」と確認する

## 引数

`$ARGUMENTS` がある場合：
- テキストがあればレポートタイトルのサフィックスとして使用（例：`/task-report Sprint2`）
- `--no-drive` フラグがあればGoogle Driveへの保存をスキップしてチャットに表示のみ

## 注意事項

- Google Drive MCPの認証が必要（`mcp__claude_ai_Google_Drive`）
- タイムスタンプが不明な場合は現在日時（`currentDate`コンテキスト）を使用
- git情報が取得できない場合はその項目をスキップする
